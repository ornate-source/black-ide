Two ideas, executed carefully

Everything else — worktrees, checkpoints, indexing — exists to support these two: nothing runs without approval, and nothing is lost.

01 · plan → approve → run

Nothing executes unreviewed

A prompt like "build a CRM" produces a plan file first. The agent stops there. You read features_plan.md, adjust it, and only then does execution start — one file at a time, one diff at a time.

✓ Manager Intake
✓ HLD & LLD Plan Created
Awaiting Approval (features_plan.md)
○ Design / Code / Test execution
02 · rollback

Undo per hunk

Every change is stored as a diff hunk, not a full file snapshot. Revert one function without touching changes elsewhere.

- old_auth_logic()
+ new_jwt_middleware()
[Revert Hunk]
03 · isolation

Isolated Git Worktrees

Manager → Architect → Design → Backend → Frontend each execute in their own git worktree branches, preventing file system and database locks when subagents edit code in parallel.

feat/api ~/.blackide/worktrees/ae3f1/
feat/ui ~/.blackide/worktrees/c94b2/
04 · retrieval

AST-Aware RAG

Context retrieval parses the Abstract Syntax Tree (AST) to chunk code at class and function boundaries, ensuring semantic searches return complete modules rather than cut-off text blocks.

// parser: ast chunking
class AuthController
└─ function verifyToken()
└─ function renewSession()
05 · zero telemetry

Compile-Time Patched

Not disabled by a simple configuration setting — telemetry endpoints are completely stripped from the VSCodium build. Extensions are resolved via the community Open VSX registry.

06 · responsiveness

Asynchronous Event Loop

The editor webview and the background agent run on decoupled event loops connected by IPC. Active agent processing or shell tasks will never freeze typing, scrolling, or editor inline chats.
